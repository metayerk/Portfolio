Programme de classification automatique de dépêche en Java

Programme de classification automatique des genres de dépêches, le programme apprend et classifier par rapport à des dépêches qui lui sont fournies leur genre. (sport, science, politique, ...)

Réalisation seul d'un programme pour déterminer automatiquement le genre d'une dépêche. Le programme fonctionne en deux temps. En premier lieu, il faut lui donner une liste de dépêches pour qu'il relie des mots-clés à un genre. Exemple, le mot "Triathlon" sera reliés au sport. Puis après avoir reçu un minimum de données, le programme va déterminer de lui-même le genre d'une dépêche en reliant les mots-clés qu'il connaît aux mots présents dans la dépêche.

En second temps, nous devions produire un rapport sur le programme montrant les fonctionnalités, puis l'analyser le coût algorithmique d'une fonction. Voici un lien de téléchargement du rapport entièrement en anglais.

Étude du Nutri-Score en langages R sur la base de données d'Openfoodfacts

Étudier l'évaluation du Nutri-Score sur les volailles française par rapport à leur composition nutritionnel, nettoyer avec PostgreSQL la base de données et produire un rapport en langages R sur notre étude.

En groupe de 2, nous avons reçu un sujet d'étude où nous devions mettre en lien la classification Nutri-Score des aliments avec leur composition nutritionnelle. Notre sujet était la volaille, pour traiter le sujet nous avons reçu un fichier CSV que nous devions insérer dans une table PostgreSQL puis en trier les données selon différents critères.

    Les critères nécessaires à l'étude :

  • Aliments : Volailles
  • Origine : France
  • Retirer les aliments mal catégorisés par les utilisateurs. En exemple, une courge notée Volailles
  • Les magasins revendeurs

Nous avons dans un premier temps produit un fichier SQL qui insère dans une base les données, puis les trie en fonction des critères résumées précédemment, enfin récupérer le fichier nettoyé avec les données utiles pour l'étude. Nombre de produits restant ~1500.

Dans un second temps, nous avons fait des statistiques en langage R pour étudier les données, puis en déduire des concordances avec l'évaluation Nutri-Score des volailles françaises.

Création et développement d'une application client-serveur sur les fond de carte historique

Installation d'un serveur web Apache2 et d'un serveur de base de données PostgreSQL, avec machine virtuel QEMU, connectés en réseau local (LAN) pour la publication d'un site dynamique sur des fonds de carte historique français.

Dans le contexte du semestre 3, notre équipe projet a été chargée de réaliser une application client-serveur pour valoriser le patrimoine culturel. Afin de répondre à cette problématique, nous allons développer un jeu web en ligne basé sur des fonds de cartes historiques. Ces cartes permettront d'explorer différentes époques de l'Auvergne-Rhône-Alpes, tout en découvrant des éléments marquants du patrimoine local.

L'objectif de notre jeu est de sensibiliser les utilisateurs, tout en leur permettant de découvrir le riche patrimoine de la région Auvergne-Rhône-Alpes. Ces derniers joueront à un jeu qui leur permettra d'explorer des fonds de cartes historiques détaillés représentant différentes périodes de l'histoire régionale. Toutes les cartes utilisées sont issues de GéoPortails. Ce jeu permettra non seulement d'enrichir leurs connaissances géographiques, mais aussi de leur faire vivre une expérience unique où histoire et découverte se mêlent à travers de notre jeu. L'application sera conçue pour être intuitive, accessible et adaptée à un large public, notamment les jeunes passionnés et les amateurs d'histoire.
